THIẾT LẬP DECAP CMS + GITHUB (Cloudflare Workers / không dùng Netlify)
=======================================================================

QUAN TRỌNG — HAI URL KHÁC NHAU
--------------------------------
- SITE URL (site đọc tin Astro): ví dụ https://astro-blog-starter-template.phucnhan218.workers.dev
  → dùng trong site_url / display_url của Decap, KHÔNG dùng làm backend.base_url.
- PROXY URL (Worker OAuth): ví dụ https://ten-worker-ban-dat.phucnhan218.workers.dev
  → CHỈ dùng cho backend.base_url. Phải deploy Worker riêng (decap-proxy).
- Lỗi DNS_PROBE_FINISHED_NXDOMAIN khi đăng nhập: thường do base_url vẫn là placeholder
  REPLACE-WITH-OAUTH-PROXY hoặc URL sai — thay bằng PROXY URL thật sau bước deploy.


1) Mở giao diện quản trị
   - Sau khi deploy site: https://<domain-site-của-bạn>/admin/


2) Worker OAuth (bắt buộc trên production) — làm theo thứ tự
   Decap không gọi GitHub API trực tiếp từ trình duyệt; cần Worker giữ CLIENT_SECRET.

   Nguồn mẫu: https://github.com/sterlingwes/decap-proxy

   A. Chuẩn bị trên máy
      - Cài Node.js, Git.
      - Đăng nhập Cloudflare CLI:  npx wrangler login

   B. Clone và cấu hình decap-proxy
      - git clone https://github.com/sterlingwes/decap-proxy.git
      - cd decap-proxy
      - cp wrangler.toml.sample wrangler.toml  (hoặc lệnh tương đương trên Windows)
      - Mở wrangler.toml: đặt `name = "ten-worker-ban-chon"` (tên này nằm trong URL workers.dev).

   C. Secrets GitHub (từ OAuth App — làm bước D trước nếu bạn muốn có Client ID/Secret sẵn)
      - npx wrangler secret put GITHUB_OAUTH_ID
      - npx wrangler secret put GITHUB_OAUTH_SECRET

   D. GitHub OAuth App (Settings → Developer settings → OAuth Apps)
      - Homepage URL: tạm thời có thể điền URL Worker sau khi biết (xem E); hoặc tạo App
        sau khi deploy và SỬA lại Homepage + Callback cho đúng.
      - Authorization callback URL:  https://<PROXY URL không có đuôi slash>/callback
        Ví dụ proxy là https://decap-oauth.phucnhan218.workers.dev
        thì callback là https://decap-oauth.phucnhan218.workers.dev/callback

   E. Deploy và LẤY PROXY URL
      - Trong thư mục decap-proxy:  npx wrangler deploy
      - Xem log: sẽ có URL dạng https://<name-trong-wrangler>.<subdomain>.workers.dev
      - Hoặc Cloudflare Dashboard → Workers & Pages → chọn Worker vừa deploy → copy URL.

   F. Cập nhật repo website (repo có Astro + public/admin/config.yml)
      - Mở public/admin/config.yml
      - Sửa:
          backend.base_url: https://<PROXY URL đầy đủ, không có /auth ở cuối>
        Giữ nguyên:
          backend.auth_endpoint: /auth
      - Commit, push, deploy lại site để /admin/ đọc config mới.


3) Khớp repo / domain
   - backend.repo, branch khớp GitHub (vd. ttttvn/astro-blog-starter-template, main)
   - site_url, display_url = URL site đọc tin (https://...), không nhầm với proxy


4) Sau khi biên tập bằng Decap
   - Decap commit lên GitHub → CI/CD / wrangler deploy build lại thì nội dung mới lên site.


5) Chỉ dùng máy, không deploy proxy (tuỳ chọn)
   - Trong public/admin/config.yml bật: local_backend: true
   - Ở thư mục gốc repo site: npx decap-server@latest
   - Mở http://localhost:4321/admin/ (cùng lúc npm run dev nếu cần)
   - Trước khi push production: tắt local_backend (comment hoặc xóa dòng).


6) Ảnh upload
   - media_folder: public/uploads — ảnh commit vào repo.


7) Chuẩn biên tập bài
   - Trên site: /huong-dan-bien-tap/
   - File nguồn: src/pages/huong-dan-bien-tap.md


Tài liệu Decap: https://decapcms.org/docs/
