THIẾT LẬP DECAP CMS + GITHUB (Cloudflare Workers / không dùng Netlify)
=======================================================================

1) Mở giao diện quản trị
   - Sau khi deploy site: https://<domain-của-bạn>/admin/

2) Đăng nhập GitHub qua OAuth (bắt buộc trên môi trường thật)
   Decap không thể gọi GitHub API trực tiếp từ trình duyệt mà không có máy chủ
   trung gian giữ CLIENT_SECRET. Cần một Worker OAuth riêng (subdomain/URL riêng).

   Khuyến nghị: dùng mẫu Cloudflare sẵn có hướng dẫn đầy đủ:
   https://github.com/sterlingwes/decap-proxy

   Tóm tắt bước (theo README của decap-proxy):
   a) Tạo GitHub OAuth App (Developer settings → OAuth Apps → New)
      - Homepage URL: URL của Worker proxy (PROXY URL), ví dụ https://xxx.workers.dev
      - Authorization callback URL: PROXY URL + /callback
        (ví dụ https://xxx.workers.dev/callback)
   b) Clone decap-proxy, tạo wrangler.toml, đặt secrets GITHUB_OAUTH_ID và GITHUB_OAUTH_SECRET
   c) npx wrangler deploy → lấy PROXY URL
   d) Sửa file public/admin/config.yml trong repo website:
        backend.base_url: <PROXY URL>
        backend.auth_endpoint: /auth
      (đã có sẵn auth_endpoint; chỉ cần thay base_url đúng Worker)

3) Sửa repo / domain trong config.yml nếu khác
   - backend.repo, branch
   - site_url, display_url (URL đầy đủ của site đọc tin, có https://)
   - backend.base_url = PROXY URL (Worker decap-proxy); backend.auth_endpoint: /auth

4) Sau mỗi lần đăng/xoá/sửa bài từ Decap
   - Decap commit thẳng lên GitHub → pipeline (Workers/Pages) cần build lại
     thì bài mới mới xuất hiện trên website.

5) Thử cục bộ không OAuth (tuỳ chọn)
   - Trong public/admin/config.yml bỏ comment dòng: local_backend: true
   - Chạy tại thư mục gốc repo: npx decap-server@latest
   - Mở http://localhost:4321/admin/ (hoặc cổng dev Astro của bạn)
   - Nhớ tắt local_backend trước khi push production nếu không dùng chế độ này.

6) Ảnh upload
   - media_folder: public/uploads — ảnh commit vào repo trong thư mục này.

Tài liệu Decap: https://decapcms.org/docs/
